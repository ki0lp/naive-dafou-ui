## 微服务入门

微服务概述

```markdown
# 微服务是一种将业务系统进一步拆分的架构风格
# 微服务强调每个单一业务都独立运行
# 每个单一服务都应该使用更轻量的机制保持通信
# 服务不强调环境，可以不同语言或者数据源
```

微服务的选择

```markdown
# dubbo
# spring cloud
# zero ice
```

微服务基本概念

```markdown
# Provider 服务提供者，提供服务实现
# Consumer 服务调用者，调用Provider提供的服务实现
- 同一个服务可以是Provider 又是Consumer
```
